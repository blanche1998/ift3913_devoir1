PARTIE 4

Les 3 classes les moins bien commentées sont celles avec le degré selon lequel une classe est bien commentée
(classe_BC) le plus faible, soit:

1) XYPlot (classe_BC = 3.70E-04)
2) CategoryPlot (classe_BC = 4.10E-04)
3) ChartPanel (classe_BC = 5.470E-04)


Les 3 paquets les moins bien commentés sont ceux avec la plus faible densité de commentaires pour un paquet (paquet_DC) soit:

1) jfreechart-m.src.main.java.org.jfree.chart (paquet_BC = 1.55E-05)
2) jfreechart-m.src.main.java (paquet_BC = 5.55E-05)
3) jfreechart-m.src.main.java.org.jfree.data (paquet_BC = 1.55E-05)


Les #2 et #3 des classes et les tops 3 packages qui ont peu de commentaires sont des classes de tests ou des packages qui ne
contiennent que des classes de tests. Plusieurs des tests n'ont aucun commentaire, donc on a aucune idée de ce qu'ils effectuent.
Il faudrait mettre des commentaires explicatifs dans ces classes afin qu'un développeur externe puisse comprendre comment ces
testsfonctionnent. 


Cependant, la densité de commentaires n'est peut-être pas une bonne métrique pour analyser si des fichiers de code sont bien
commentés. En effet, cette métrique ne permet pas de prendre en compte la pertinence des commentaires ou le fait qu'il puisse
y avoir trop de commentaires, rendant le fichier de code excessivement dense et difficile à comprendre.
